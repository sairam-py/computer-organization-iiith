Tiling Matrix Multiplication

Introduction

There are three possible kinds of cache misses.

  1. Compulsory misses
  2. Conflict misses
  3. Capacity misses

Compulsory misses can possibly be circumvented by using data prefetching techniques. Conflict misses can be taken care by increasing the associativity of the cache architecture. Whereas capacity misses can be taken care of by using program transformations which decrease the working set size of the running program. In this experiment we shall understand a program transformation called loop tiling and its impact on the working set size of a program.

Theory

Capacity misses arise when the size of the working set of a program is greater than the size of the cache. Changing the cache line size or associativity will not have an effect on capacity misses. We have to resort to program transformations which reduce the working set size. A classical example of such a transformation is loop tiling which can be best understood by using the matrix multiplication program as an example. In this example, we divide the two matrices to be multiplied into tiles and perform multiplication over tiles. The effectiveness of this approach in decreasing the capacity misses depends on the tile size. However the optimal tile size is a function of the underlying cache architecture. In this experiment, we can analye the impact of the cache architecture on the the optimal tile size.

Objective

The objective of this experiment is to understand how we can use loop tiling transformation to decrease the working set size of a program.
